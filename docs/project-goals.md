# 房租管理系統專案目標

## 專案願景

建立一套以屋主收租管理為核心的房租管理系統，協助屋主或管理者集中管理多位屋主、多個大樓社區、多間房屋、多位租客、租約歷史、每月租金帳款、付款紀錄與未繳提醒。

系統的核心目標不是單純記帳，而是完整追蹤：

- 哪位屋主擁有哪些房屋
- 房屋位於哪個大樓或社區
- 哪位租客目前租住哪間房屋
- 每份租約的租金、押金、租期與繳租日
- 每月租金是否已繳
- 未繳或逾期租金是否已提醒租客

## 需求背景

本系統需支援以下實務情境：

1. 一位屋主可以擁有多間房屋。
2. 房屋可能位於不同的大樓或社區。
3. 系統中可能有多位不同屋主。
4. 租客可能從一間房屋移動到另一間房屋。
5. 租金可能在重新簽約或續約時調漲。
6. 每月需自動提醒尚未繳房租的租客。

## 核心設計原則

### 租客不直接綁定房屋

租客不應直接儲存在房屋資料上，而應透過「租約」與房屋建立關聯。

這樣可以支援租客搬家、換房、退租後再承租等情境，並且保留完整歷史紀錄。

### 租金不直接綁定房屋

租金不應只存在房屋資料上，而應存在「租約」上。

這樣當租金在重新簽約或續約時調整，系統可以保留過去租約的租金，不會覆蓋歷史資料。

### 每月租金帳款獨立產生

每月應依據生效中的租約產生租金帳款，並追蹤該月份的應繳金額、已繳金額、到期日與繳款狀態。

## 核心資料模型

### 屋主 Owner

用來管理房屋所有權人。

主要欄位：

- 姓名
- 聯絡電話
- Email
- 地址
- 備註

關係：

- 一位屋主可以擁有多間房屋。

### 大樓或社區 Building

用來管理房屋所在的大樓、社區或集合住宅。

主要欄位：

- 名稱
- 地址
- 管理單位
- 備註

關係：

- 一個大樓或社區可以包含多間房屋。

### 房屋 Property

用來管理可出租的房屋或單位。

主要欄位：

- 所屬屋主
- 所屬大樓或社區
- 門牌地址
- 樓層
- 房號
- 房型
- 坪數
- 狀態：空屋、出租中、停用
- 備註

關係：

- 一間房屋屬於一位屋主。
- 一間房屋可位於一個大樓或社區。
- 一間房屋在不同時間可以有多份租約。

### 租客 Tenant

用來管理承租人的基本資料。

主要欄位：

- 姓名
- 電話
- Email
- LINE ID
- 身分證字號
- 緊急聯絡人
- 備註

關係：

- 一位租客可以有多份歷史租約。
- 一位租客可以從一間房屋搬到另一間房屋。

### 租約 Lease

用來連結租客、房屋與租金條件。

主要欄位：

- 租客
- 房屋
- 起租日
- 結束日
- 月租金
- 押金
- 每月繳租日
- 合約狀態：生效中、即將到期、已結束
- 備註

關係：

- 一份租約屬於一位租客。
- 一份租約對應一間房屋。
- 一份租約可以產生多筆每月租金帳款。

### 租金帳款 Rent Invoice

用來記錄每個月份的應收租金。

主要欄位：

- 對應租約
- 帳款月份
- 應繳金額
- 已繳金額
- 到期日
- 狀態：未繳、部分繳、已繳、逾期
- 備註

關係：

- 一筆租金帳款屬於一份租約。
- 一筆租金帳款可以有多筆付款紀錄。
- 一筆租金帳款可以有多筆提醒紀錄。

### 付款紀錄 Payment

用來記錄租客實際付款狀況。

主要欄位：

- 對應租金帳款
- 付款日期
- 付款金額
- 付款方式：現金、轉帳、支票、其他
- 收據編號
- 備註

### 提醒紀錄 Reminder

用來記錄系統對未繳或逾期租金的提醒歷史。

主要欄位：

- 對應租金帳款
- 租客
- 提醒時間
- 提醒方式：Email、簡訊、LINE、系統通知
- 發送狀態：待發送、成功、失敗
- 失敗原因

## 主要功能

### 儀表板

提供屋主或管理者快速掌握收租狀況。

功能包含：

- 本月應收租金
- 本月已收租金
- 本月未收租金
- 逾期帳款數量
- 未繳租客列表
- 即將到期租約

### 屋主管理

功能包含：

- 新增、編輯、刪除屋主
- 查看屋主名下房屋
- 查看屋主收租統計
- 查看屋主相關租約與帳款

### 大樓或社區管理

功能包含：

- 新增、編輯、刪除大樓或社區
- 查看社區底下房屋
- 查看各社區出租狀態

### 房屋管理

功能包含：

- 新增、編輯、刪除房屋
- 指定房屋所屬屋主
- 指定房屋所屬大樓或社區
- 查看目前出租狀態
- 查看目前租客
- 查看歷史租約

### 租客管理

功能包含：

- 新增、編輯、刪除租客
- 查看租客目前租住房屋
- 查看租客歷史租屋紀錄
- 查看租客付款紀錄

### 租約管理

功能包含：

- 建立新租約
- 結束租約
- 續約
- 重新簽約
- 調整租金
- 查看生效中租約
- 查看即將到期租約
- 查看已結束租約

### 收租管理

功能包含：

- 每月自動產生租金帳款
- 查看本月帳款
- 查看未繳帳款
- 查看逾期帳款
- 登錄付款
- 支援部分付款
- 更新帳款狀態

### 自動提醒

功能包含：

- 每日排程檢查未繳帳款
- 到期日前提醒租客
- 到期日當天提醒租客
- 逾期後定期提醒租客
- 記錄每次提醒結果

## 開發階段

### 第一階段：基礎資料管理

建立核心資料模型與基本管理功能。

範圍包含：

- 屋主管理
- 大樓或社區管理
- 房屋管理
- 租客管理
- 租約管理

### 第二階段：收租流程

建立租金帳款與付款紀錄。

範圍包含：

- 每月租金帳款產生
- 租金帳款列表
- 付款登錄
- 未繳、部分繳、已繳、逾期狀態判斷

### 第三階段：租約歷史與異動

支援實務租屋異動情境。

範圍包含：

- 租客換房
- 租約續約
- 重新簽約
- 租金調漲
- 歷史租約查詢

### 第四階段：提醒與通知

建立未繳租金提醒機制。

範圍包含：

- 自動檢查未繳租金
- 到期日前提醒
- 逾期提醒
- 提醒紀錄查詢
- 發送失敗重試

### 第五階段：報表與進階功能

提升管理效率與資料可用性。

範圍包含：

- 屋主收租報表
- 大樓出租率報表
- 房屋收益統計
- 租客付款紀錄匯出
- Excel 匯出
- 權限管理

## 初步資料關係

```text
屋主 Owner
  -> 房屋 Property

大樓或社區 Building
  -> 房屋 Property

房屋 Property
  -> 租約 Lease
  -> 租客 Tenant

租約 Lease
  -> 租金帳款 Rent Invoice

租金帳款 Rent Invoice
  -> 付款紀錄 Payment
  -> 提醒紀錄 Reminder
```

## 成功標準

本系統完成後，應能達成以下目標：

- 管理多位屋主與多間房屋。
- 清楚知道每間房屋目前是否出租。
- 清楚知道每位租客目前承租哪間房屋。
- 保留租客搬家、退租、重新簽約與租金調整的歷史紀錄。
- 每月自動產生應收租金。
- 快速查詢未繳與逾期租金。
- 自動提醒尚未繳租的租客。
- 提供屋主收租狀況與房屋收益的基礎報表。
