# 房租管理系統 UI/UX 設計計畫

## 設計目標

本系統的 UI/UX 以屋主與房屋管理者的日常收租流程為核心，目標是讓使用者能快速掌握出租狀態、收租進度、未繳租客、逾期帳款與租約異動。

介面應具備以下特性：

- 資訊清楚，可快速掃描。
- 操作路徑短，適合每月重複使用。
- 以資料表、篩選、狀態標籤與摘要數字為主。
- 避免行銷式首頁，登入後第一畫面即為可操作的管理儀表板。
- 支援桌機優先，同時兼顧平板與手機查詢。

## 主要使用者角色

### 系統管理者

負責管理所有屋主、大樓、房屋、租客、租約與收租資料。

常見任務：

- 建立屋主資料
- 建立房屋資料
- 新增租約
- 檢查每月收租狀態
- 處理逾期帳款
- 查看報表

### 屋主

查看自己名下房屋與收租狀態。

常見任務：

- 查看本月應收與已收金額
- 查看名下房屋出租狀況
- 查看未繳租客
- 查看租約與付款紀錄

### 收租管理者

負責日常收租、登錄付款與發送提醒。

常見任務：

- 查看本月未繳清單
- 登錄租客付款
- 發送或重送提醒
- 查詢付款紀錄

## 核心頁面設計

### 1. 儀表板

目的：讓使用者一進系統就知道本月收租狀況。

主要區塊：

- 本月未收

- 今日待處理提醒

- 最近付款紀錄

- 查看未繳清單
- 發送提醒
- 切換月份

## 主要操作流程

### 新增房屋流程

```text
進入房屋管理
  -> 點擊新增房屋
  -> 選擇屋主
  -> 選擇大樓或社區
  -> 輸入房屋資料
  -> 儲存
  -> 房屋狀態預設為空屋
```

### 建立租約流程

```text
進入租約管理或房屋明細
  -> 點擊建立租約
  -> 選擇租客
  -> 選擇房屋
  -> 輸入起租日、到期日、月租金、押金、繳租日
  -> 儲存
  -> 房屋狀態更新為出租中
  -> 系統準備依租約產生每月租金帳款
```

### 租客換房流程

```text
進入租客明細
  -> 查看目前租約
  -> 結束目前租約
  -> 建立新租約
  -> 選擇新房屋
  -> 輸入新租約條件
  -> 儲存
  -> 保留舊租約歷史
```

### 租金調漲流程

```text
進入租約明細
  -> 點擊續約或重新簽約
  -> 輸入新租期
  -> 輸入新月租金
  -> 儲存
  -> 舊租約結束
  -> 新租約生效
  -> 歷史租金保留在舊租約
```

### 每月收租流程

```text
系統依生效租約產生本月租金帳款
  -> 管理者進入收租管理
  -> 篩選未繳或逾期
  -> 登錄已付款租客
  -> 對未繳租客發送提醒
  -> 系統記錄付款與提醒紀錄
```

## 表單設計原則

- 必填欄位需清楚標示。
- 金額欄位需使用千分位顯示。
- 日期欄位使用日期選擇器。
- 屋主、大樓、房屋、租客使用可搜尋下拉選單。
- 儲存前需顯示必要驗證錯誤。
- 刪除或結束租約等高影響操作需二次確認。

## 資料表設計原則

- 支援搜尋、篩選、排序與分頁。
- 狀態使用顏色標籤輔助辨識。
- 金額欄位靠右對齊。
- 日期欄位格式一致。
- 每列右側提供常用操作。
- 手機版以卡片式清單呈現。

## 狀態與色彩語意

建議使用克制、清楚的管理系統色彩。

- 已繳：綠色
- 未繳：灰色或藍灰色
- 部分繳：黃色
- 逾期：紅色
- 即將到期：橘色
- 空屋：藍色
- 停用：灰色

整體視覺應避免過度鮮豔或單一色系堆疊，保留足夠對比，讓使用者能快速辨識風險與待辦事項。

## 權限與可見性

不同角色看到的資料範圍應不同。

- 系統管理者可查看全部資料。
- 屋主只能查看自己名下房屋、租約與收租資料。
- 收租管理者可查看與處理收租、付款、提醒資料。

## 響應式設計

### 桌機

- 主要支援完整資料表與多欄資訊。
- 適合批次查詢、篩選與管理操作。

### 平板

- 保留側邊導覽，但可收合。
- 資料表可橫向捲動。
- 明細可使用右側抽屜面板。

### 手機

- 導覽改為底部導覽或漢堡選單。
- 資料表改為卡片清單。
- 常用操作固定於底部。
- 優先支援查詢、付款登錄與提醒發送。
